如何提高 ssh 的速度?

2020-11-11 Linux 使用 PV:

最近一直是 ssh 开发, 使用 vscode 开发没什么延迟的感受, 但当我终端 ssh 连接总是不太流畅.

ssh 多路复用

  • ssh 多路复用指的是: 我们可能会频繁产生ssh连接, 当我们退出某次ssh时, 无需真的退掉会话, 下一次连接复用即可. 大概是那种池化的思想…

ssh 压缩传输

  • ssh 这种模型大概看起来有点类似于ping pong 模型.
  • ssh -v 可以看到: 有一个大写的 C 选项, 表示与远程的服务器的数据传输压缩传输, 看似是减少了带宽, 但实则没什么用, 大概适合于那种延迟低, 宽带速度有限的那种网络环境(例如公共网络, 办公, 宿舍?).

罪魁祸首

  • 我使用的是 zsh, 上面还用了一些主题和插件设置
  • 一些带有状态的插件, 每当我输入命令, 其实他们都要执行一次, 例如 git plugin, 还有一些计时的插件, 等等这些其实都会影响使用.
  • 减少涉及到状态的插件使用, 可以显著加速. >_< 所以使用bash 是最快的吗?(滑稽)
