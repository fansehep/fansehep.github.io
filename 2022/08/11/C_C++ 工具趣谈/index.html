C, C++ 工具趣谈

2022-08-11 C++tools PV:

引言

自由大概是 C++ 的一大特色, 标准委员会们不负责编写编译器代码, 所以 C++ 在不同的平台不同的编译器的标准库实现都不太一样, 但只需保证接口与标准相同即可. 本文简单介绍笔者用过的几款C++工具. 又因为缺乏包管理器, 所以C++的构建工具各自为家. 大概以后会对包管理器指定一个标准???

Bazel

Bazel 是谷歌开源出来的一款构建套件, 笔者曾用过该构建工具一段时间, 使用 skylark 这种描述性的语言来描述构建规则, 十分优雅. 容易上手, 且拥有强大的生态, 谷歌出手, 必属精品.

Bazel doc link

CMake

CMake 老熟人了, 只是笔者个人认为他的语法实在是太过于丑陋, 对其实在谈不上喜欢.

CMake doc link.

C / C++ 补全

笔者建议使用 clangd 进行构建, 该插件在vscode, vim 等等都有支持. (clangd 与 微软C++ 扩展冲突, 直接卸载即可).

  • vscode clangd extension
  • vim clangd plugin

clangd 进行补全的前提是请在项目下已有 compile_commands.json 文件, 用于 clangd 进行识别.

生成 compile_commands.json

  • Bazel 目前官方还没有实现 compile_commands.json 的生成, 但 https://github.com/grailbio/bazel-compilation-database 该项目可以做到.
  • CMake 已经与bear合作, 只需在命令行中添加-DCMAKE_EXPORT_COMPILE_COMMANDS=on 即可. 更多详见 CMake Compile Doc
  • Makefile
    • Makefile 推荐使用 python 编写的 compiledb 工具可以直接生成.

clang-tidy

静态分析代码也是 C++ 的一大痛点, clang-tidy 只需要在你的根目录下指定检查规则, 便能及时对你的代码进行分析, 详见 clang-tidy doc.

clang-format

用于快速统一格式化代码的利器, 详见 clang-format doc.

colorful comments

vscode 的一款彩色注释插件, link.

总结

所有的 IDE 都应该内置一款编译时小游戏 或者 推送几篇blog(尤其是 C++), 不然不是一个合格的 IDE.😘

趣谈

与 rust, go 这些语言不同, C++ 是一个很自由的语言, C++ 语言的进化主要依靠标准委员会进行投票来发布新标准, 等待各大编译器厂家实现.
关于 不干正事的标准委员会, 这里C++ 标准委员会 写的不错.

总的来说, 笔者个人不太看好这种语言被多方志愿者所共同管理的形式.

语言进化速度比较慢, 希望包管理器和标准网络库能快速出来.

不过, 对于标准网络库, Cpper 真的会用它而不是造轮子吗?🥵
